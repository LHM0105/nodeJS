关于node

基于chrome v8引擎的 以事件驱动和非阻塞I/O流 构建的模型 ，轻量 高效（node中使用的语言是javascript）

npm是目前 世界上 最大的 包管理器

node 基于commonJS规范

amd 异步加载规范 提前引入，代表框架requireJS

cmd 通用加载规范 按需引入，代表框架seaJS

commonJS 按需加载，核心：接收和暴露

接受需要用require

暴露用module.exports

}

关于node中的api

url模块：

方法

parse|urlStr parseQueryString slashesDenoteHost 将路径字符串解析为路径对象

urlStr 要解析的路径字符串

parseQueryString 布尔类型，主要用来解析query，设置为true时，query一项的值是对象格式

slashesDenoteHost 布尔类型，当不确定要请求的协议时，辅助进行解析，设置为true可以帮助解析

format|urlObj parseObj slashesObj 将路径对象转化为路径字符串

urlObj 要解析的路径对象

resolve | from to 将基础路径和后缀路径转为目标路径

注意：基本路径要在最后添加‘/’，否则合并会找到最近的‘/’并替换其后面的内容

queryString模块：为查询字符串提供扩展

方法

parse|str con seq 将查询字符串转为对象，参数与stringify相同

str 查询数据 字符串

stringify|strObj con seq 将查询数据对象转为字符串

strObj 查询数据 对象

con 键值对之间的分隔符，默认是“&”

seq 键与值之间的分隔符，默认为“=”

escape 相当于encodeURI 将Asc编码转换成utf-8

unescape 相当于decodeURI 将utf-8转为Asc编码

events模块 对事件进行处理

方法

在node中的事件我们使用on或者once进行监听

往往配合推送emit去使用

once 只能监听一次；on 时刻进行监听

fs(filesystem)模块 文件管理系统

方法

stat(path,callback(err,statsdata){}) 获取与文件相关的信息

mkdir(path,callback(err){}) 新增目录

writeFile(path,con,callback(err){}) 新增文件

appendFile(path,con,callback(err){})追加内容

readFile(path,charset,callback(err,data){}) 读取文件内容

resddir(path,callback(err,filedata){}) 读取文件目录

rmdir(path,callback(err){}) 删除目录

*空文件夹才能被删除，所以在删除文件夹前需要把其中的文件全部删除

fs.readdirSync("img").map((file) => { fs.unlink(`img/${file}`,(err) => { console.log("删除文件夹中所有文件"); }); });

unlink(path,callback(err){}) 删除文件
